遍历器遍历(Iterator)(一般不直接用)

支持遍历器的数据结构,具有Symbol.iterator属性.调用该接口可以返回一个遍历器对象

原生支持的数据结构

  1. 数组
  2. Map
  3. Set
  4. 字符串
  5. TypedArray
  6. 类数组对象中的arguments, NodeList对象
  7. Generator对象

遍历方式

为对象部署遍历器方法

遍历器接口被使用的场景

  1. 数组,Set的解构赋值
  2. 扩展运算符(…) 数组,字符串,Set,Map
  3. yield*, 会调用后面跟的数据结构的遍历器接口
  4. 接受数组作为参数的场合
    • for … of 最常用的遍历方式之一
    • Array.from()
    • Map(),Set(),WeakMap(),WeakSet() 注意此处说的是构造函数, WeakMap和WeakSet本身是不可遍历的.
    • Promise.all()/race()

for … of 循环(利用了iterator,推荐)

优点: 写法简单, 且可以配合break, continue, return 使用,提供了遍历所有数据结构的统一操作接口(iterator)

for … in 循环

优点: 写法简单

缺点: 数组的键名是数字,但会转成字符串; 数组上的属性也会被遍历出来, 甚至包括原型链上的键(这通常是不被期待的), 某些情况下会以任意顺序遍历键名

支持的数据结构

  1. 数组(不建议使用, 见上面缺点)
  2. 对象

原始for循环

优点: 操作可控,直观

缺点: 写法繁琐

支持的数据结构

  1. 数组
  2. 字符串(不建议使用, 无法正确识别码点超过\uFFFF的字符!)

forEach循环

优点: 写法简单

缺点: 无法中途跳出循环

支持的数据结构

  1. 数组
  2. Map
  3. Set

ES6为止, 所有的遍历方式

字数统计: 1k 字 | 阅读时长 ≈ 4 分钟

遍历Set. 例子: let set = new Set([‘red’, ‘green’, ‘blue’]);

  • 遍历器生成函数遍历

    • keys() 返回键名的遍历器

    • values() 返回键值的遍历器,与keys()方法完全一致 (Set的默认遍历器)(Object.getPrototypeOf(set)[Symbol.iterator] === set.keys)

    • entires() 返回键值对的遍历器

    注: 扩展运算符(…)内部使用的就是for …of 循环

  • forEach() 使用回调函数遍历每个成员

    注: 第三个对象都是该set对象

  • 转为数组, 用数组的方式遍历

    […set] (values())

遍历Map. 例子: let map = new Map([[‘F’, ‘no’], [‘T’, ‘yes’]]);

  • 遍历器生成函数遍历

    • keys() 返回键名的遍历器

    • values() 返回键值的遍历器

    • entires() 返回键值对的遍历器 (Map的默认遍历器)(Object.getPrototypeOf(map)[Symbol.iterator] === map.entries)

    注: 扩展运算符(…)内部使用的就是for …of 循环, 另外解构赋值, Array.from方法内部都调用了遍历器接口

  • forEach() 使用回调函数遍历每个成员

    注: 第三个对象都是该map对象

  • 转为数组, 用数组的方式遍历

    […map] (entries())

遍历数组.(此处没包括数组的map, filter等遍历方法) 例子: arr[‘a’, ‘b’, ‘c’]

  • for循环遍历

  • 遍历器生成函数遍历

    • keys() 返回键名的遍历器

    • values() 返回键值的遍历器(数组的默认遍历器)(Object.getPrototypeOf(arr)[Symbol.iterator] === arr.values)

    • entires() 返回键值对的遍历器

    注:数组遍历器遍历只返回具有数字索引的属性

  • forEach遍历

  • for … in 循环遍历,只读取键名

    注: 数组for … in 循环会读取所有键名

遍历字符串. 例子: str = ‘Hello’

  • for循环遍历(存在问题, 不推荐)

    注: 存在无法正确识别码点超过\uFFFF的字符的问题!

  • 遍历器遍历

    注: 字符串的for … of循环可以正确识别32位的UTF-16字符.

遍历类数组对象,arguments, NodeList

  • 遍历器遍历

    注: 不是所有类数组对象都部署了遍历器, 如需遍历,使用Array.from()方法,转换为数组后遍历.

遍历对象 例子: obj = {name : ‘zs’, age: 12};

  • 其他可行方案

为什么会如此呢:

js代码是单线程执行的.js代码分同步代码和异步代码,同步代码执行先于异步代码,线程会先执行同步代码.当异步代码满足了执行条件,会进入执行队列中, 当线程执行完同步代码, 线程会去找异步代码的执行队列执行队列任务.
